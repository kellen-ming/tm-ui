## 怎么理解这组 API 的配合关系

可以把它们理解成一条很清晰的链路：

1. `defineDocs` 负责告诉系统“内容在哪”
2. `defineConfig` 负责告诉系统“怎么处理内容”
3. `loader` 负责把这些内容转成“可查询、可生成路由的 source”
4. `DocsLayout` 和 `DocsPage` 负责把这些内容最终显示出来

## 推荐实践

<Callout type="success" title="建议">
  把“内容定义”和“页面渲染”分开维护。前者放在 `content/docs` 和 `source.config.ts`，后者放在 `app/docs` 和自定义 docs 组件里。
</Callout>
