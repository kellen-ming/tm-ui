---

## 适用场景

这套快速入门特别适合下面这类项目：

- 已经有一批组件和 demo
- 想保留原有首页或 showcase
- 想新增 `/docs`，而不是重写整个站点

## 最小闭环

<Steps>
  <Step>

### 1. 安装依赖

```bash
pnpm add fumadocs-ui fumadocs-core fumadocs-mdx @types/mdx
```

  </Step>
  <Step>

### 2. 配置 MDX 入口

把 `next.config` 改成 `next.config.mjs`，并通过 `createMDX()` 包装 Next.js 配置。

  </Step>
  <Step>

### 3. 定义 docs source

创建 `source.config.ts` 和 `lib/source.ts`，分别负责内容集合定义与 source 封装。

  </Step>
  <Step>

### 4. 新增 docs 路由

创建 `app/docs/layout.tsx` 和 `app/docs/[[...slug]]/page.tsx`，让文档站和原有首页并存。

  </Step>
  <Step>

### 5. 写第一篇样板文档

先从一个最简单的组件开始，比如 `Button`，优先跑通标题、描述、代码示例和最小 preview。

  </Step>
</Steps>

## 一个实用建议

<Callout type="warn" title="不要一开始就上 Chatbot">
  更高效的顺序是：先把 docs 骨架搭起来，再补内容和搜索，最后再让 AI 基于真实文档工作。
</Callout>

## 下一步建议

当快速入门跑通之后，建议马上做两件事：

1. 迁移第二个组件文档页
2. 把现有的 props controls 和代码复制能力接回 docs 页面
