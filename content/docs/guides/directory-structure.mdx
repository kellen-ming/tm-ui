## 推荐理解方式

先把目录分成四层：

- 路由层
- 内容层
- 组件层
- source 层

## 每个目录分别负责什么

<Accordions type="single" collapsible>
  <Accordion title="app/docs">
    负责文档站的路由层。`layout.tsx` 是 docs 外壳，`[[...slug]]/page.tsx` 是具体页面渲染器。
  </Accordion>
  <Accordion title="content/docs">
    负责真正的文档内容。以后新增页面，优先在这里写 MDX，而不是先写 React 页面。
  </Accordion>
  <Accordion title="components/docs">
    放文档专用的自定义组件，比如 demo、说明块、交互示例。适合承载比 MDX 更复杂的 UI。
  </Accordion>
  <Accordion title="lib/source.ts 与 source.config.ts">
    一个负责定义 docs 集合，一个负责把它转成 source 能力，二者共同构成 docs 的数据入口。
  </Accordion>
</Accordions>

## 什么时候需要新建自定义组件

当出现这些情况时，建议从 MDX 抽成 `components/docs/*`：

- 页面里有复杂交互
- 同一段内容会被多个页面复用
- 需要和现有业务组件联动
- 纯 MDX 已经让页面变得难维护
