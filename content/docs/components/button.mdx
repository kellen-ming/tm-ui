Displays a button or a component that looks like a button.

Button 是 TM UI 中最基础也最常用的交互组件之一，适合承载页面中的主要操作、次级操作以及轻量级文本跳转。

## Installation

TM UI 已内置 Button 组件，直接按路径导入即可。

```tsx
import { Button } from "@/components/ui/tm-button";
```

组件源码位于 `components/ui/tm-button.tsx`，如果后续要统一调整视觉层级、尺寸或状态样式，这个文件会是主要维护入口。

## Usage

```tsx
import { Button } from "@/components/ui/tm-button";

export function Example() {
  return <Button>Primary</Button>;
}
```

基础场景建议优先从 `primary`、`secondary`、`text` 和 `link` 这四类开始选型，它们基本覆盖了大多数内容页和表单场景。

推荐使用方式：

- `primary`：页面中的主 CTA，适合一个区域内最重要的操作
- `secondary`：与主按钮并列出现的次级操作
- `text`：信息密度较高的模块中使用的低强调操作
- `link`：更接近正文语义的跳转入口
- `linear`：更适合营销页、Hero 区域和需要更强视觉吸引力的操作
- `ghost`：适合中性容器中的悬浮型补充操作
- `*-white`：用于深色背景、图片背景或品牌色块背景

## API Reference

### Button

`Button` 是对原生 `button` 元素的一层样式封装，同时支持通过 `asChild` 透传给链接或其他交互元素。

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `children` | `ReactNode` | required | 按钮内部内容，通常是文本、图标或两者组合。 |
| `variant` | `"primary" \| "primary-white" \| "secondary" \| "secondary-white" \| "text" \| "text-white" \| "link" \| "link-white" \| "linear" \| "ghost"` | `"primary"` | 控制按钮的视觉层级和表面样式。 |
| `size` | `"sm" \| "lg" \| "icon"` | `"lg"` | 控制按钮尺寸。`icon` 适用于仅图标按钮。 |
| `asChild` | `boolean` | `false` | 启用后不会直接渲染 `button`，而是把样式和行为透传给子元素。 |
| `type` | `"button" \| "submit" \| "reset"` | browser default | 表单场景建议显式声明。 |
| `disabled` | `boolean` | `false` | 禁用按钮交互，并自动应用禁用态样式。 |
| `className` | `string` | - | 用于叠加额外样式类名，适合处理局部布局或特殊造型。 |

## Notes

<Callout type="warn" title="表单场景建议">
  在表单或弹窗里使用按钮时，建议显式传入 `type="button"`、`type="submit"` 或 `type="reset"`，避免依赖浏览器默认行为。
</Callout>

- 一个内容区域内通常只保留一个 `primary` 按钮，避免视觉主次混乱
- `*-white` 变体应放在深色或高对比背景上，避免边界和文本层级不清晰
- 图标按钮必须提供清晰的 `aria-label`
- 如果只是页面内的文字跳转，优先考虑 `link` 或 `text`，不要滥用高强调按钮
- 使用清晰的动词文案，例如“提交申请”“查看详情”“继续下一步”
